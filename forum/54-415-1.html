Y.O.B.A. сканер
Fr0do Дата: Воскресенье, 07.07.2013, 08:25 | Сообщение # 1
Сообщений: 274
Статус: Offline
 Youth-Oriented-and-Bydlo-Approved прога для быстрого многопоточного перебора доменных имён и IP-адресов. Может ощутимо помочь в поиске некэшированных сайтов и FTP-хранилищ со свободным доступом.




Принцип работы:
1. Последовательно перебирает доменные имена или IP-адреса по заданной маске и пингует их.
2. Опционально пробует загрузить главную страницу (HTTP) или подключиться к FTP (порт 21) без логина и пароля.
3. Обнаруженные действующие сайты заносятся в отчёт и могут быть открыты в браузере.
Содержимое стандартных ini-файлов:
tlds.ini — Перечень доменов верхнего уровня, которым заполняется выпадающий список в проге.
chars.ini — Символы, из которых генерируются доменные имена: английский алфавит, цифры и дефис. Можно отредактировать, оставив, к примеру, только цифры.
Скрытые нерегулируемые параметры:
Таймаут пинга = 1000мс.
Таймаут установления связи с FTP или загрузки HTML-страницы = 10 секунд.

- Прога испытывает проблемы с подключением, если Internet Explorer находится в автономном режиме. В Windows 7 при этом невозможно сканить IP с проверкой главной страницы.
- Сканирование ведётся только вверх по диапазону и остановить его можно только вручную.
- Не стоит устанавливать флажок "открывать в браузере" при сканировании плотно заселённых диапазонов (типа *****.com) на высоких скоростях — напор в полсотни ежесекундно открывающихся вкладок браузер не выдержит :3
- На системах с мощным процессором, но медленным интернет-каналом (< 1 мегабит/с) лучше не использовать скорость "100% загрузка ЦП", иначе очередь запросов будет переполняться и уменьшит эффективность скана или даже обрушит прогу.
+ Прога отображается поверх всех окон, не давая браузеру перехватывать фокус в момент открытия обнаруженных сайтов, благодаря чему сканирование всегда можно остановить одним нажатием.
* Некоторые элементы интерфейса снабжены всплывающими подсказками.

Используя прогу для перебора IP-адресов:
Желательно снять флажок "игнорировать не пингующиеся", чтобы не пропускать айпишники, которые не откликаются на пинг, но содержат HTML. При этом автоматически установится флажок "проверять наличие главной страницы скачиванием", устраняющий ложные срабатывания на "пустышки" — пингуемые IP без HTML-страниц. %%О как!%%
Нежелательно выставлять скорость "100% загрузка ЦП" — сканирование может прерваться (прозреваю провайдерскую защиту от DDoS-а).
Очевидное: IP-адрес состоит из 4х групп, поэтому должны быть активны только 4 счётчика (раскрывающихся списка) и нет необходимости в суффиксах/префиксах и tlds.ini. Хороший полигон для обучения и тестирования возможностей проги — диапазон IP гугла, начиная с 74.125.232.0.

CHANGELOG для версии №2:
+ Упрощён интерфейс (Win7 -> 9x) с целью повышения производительности и утоления бархата у некоторых анонов.
+ Новый предел длины для сканируемой части доменного имени — 6 символов. 8 было явным излишеством, ведь полный "прочёс" одних только 6-символьных комбинаций занимает больше месяца (из цифр — полчаса).
+ Ликвидирована чудовищная утечка памяти. Введено ограничение на 10000 активных потоков (40-60Мб RAM), при превышении самые старые потоки принудительно завершаются.
+ Штатное закрытие проги больше не вызывает критическую ошибку.
+ Устранён вылет, вызываемый синхронным обращением к логу из разных потоков. Теперь прога стабильна даже при быстром скане в "густонаселённых" диапазонах.
+ Добавлена кнопка СБРОС, обнуляющая прогресс сканирования.
+ Ручное обновление статистики по клику.
+ Адрес заносится в лог вместе с протоколом http:// либо ftp://.

CHANGELOG для версии №3:
+ Возможность сохранения главных страниц со всех обнаруженных сайтов для исследования их содержимого в оффлайне. Сохраняется только HTML, без картинок, стилей, скриптов и т.п.
+ Новая опция — перебор доменов верхнего уровня, придаёт скан-последовательности вид: aaa.com/aaa.net/aaa.org/…/aab.com/aab.net/aab.org/…
+ Запрет запуска нескольких копий программы (защита от перегрузок на всякий пожарный).
+ Защита от потери наработанного при случайном закрытии окна — после нажатия на [x]или Alt+F4 запрашивается подтверждение. Через диспетчер задач прога по-прежнему выносится мгновенно :3
+ Статистика сканирования обновляется автоматически не реже раза в секунду.
+ Элементы интерфейса сгруппированы в 3 блока — формат имени, опции сканирования и журнал со статистикой. Некоторые надписи окрашены для большей наглядности.
+ Много мелких поправок и дополнений — увеличенные поля ввода, более полное обнуление статистики, новые всплывающие подсказки, доменные зоны .eu и .co.uk в tlds.ini и т.д.

CHANGELOG для версии №4:
+ Автоматическое удлинение сканируемой части имени после достижения крайних значений всеми символами (…/8/9/-/aa/ab/ac/…). Устраняет излишнюю зацикленность скана.
+ Возможность формирования имени с взаимной перестановкой префикса и суффикса (в дополнение к обычному) — даёт результативность 2х сканов за один запуск, поскольку не приходится вручную менять местами префикс/суффикс и запускать ещё один скан. %%Не особо важная функция, но может когда-нибудь пригодиться.%%
+ Опция записи в лог размеров и заголовков загружаемых страниц (так легче судить об их содержимом). Заголовки извлекаются только из тегов <title></title> с буквами в нижнем регистре, <TITLE></TITLE> не поддерживаются. Кодировка UTF-8 "кастрируется" до ANSI при занесении в лог, поэтому заголовки со всяческими иероглифами отображаются в виде "???????".
+ Чтобы не пропускать IP, которые не откликаются на ping, но содержат HTML, теперь можно отключать предварительный отсев пингом. При этом автоматически включается проверка главной страницы. Снимать флажок "игнорировать не пингующиеся" имеет смысл только при IP-скане.
+ Кнопка для открытия любого выбранного адреса из лога.
+ Кнопка для быстрой проверки на гуглимость содержимого сайта — запрашивает страницу "http://www.google.com/search?q=site:адрес_проверяемого_сайта" и при обнаружении в ней блока "search…did not match any documents" считает сайт некэшированым.
